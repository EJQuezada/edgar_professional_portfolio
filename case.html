Case study for myFlix Full-Stack Web Development Project

Overview

myflix is a web application, developed using the MERN stack, that provides users with access to information about movies, directors, and genres. Users are able to create an account, update their personal data, and create a list of favorite movies from the database.


Purpose & Content

myFlix was a personal project that I built as part of my Web Development course at CareerFoundry to showcase my mastery of full-stack JavaScript development.


Objective

The goal of the project was to create a comprehensive full-stack project that I can include in my professional portfolio. For this purpose, I built the complete server-side and client-side for the application from scratch.

Approach

Server-Side

I created a RESTful API using Node.js and Express that interacts with a non-relational database on MongoDB. The API is accessible via commonly used HTTP methods like GET or POST. In order to retrieve data from and store data into the database, CRUD methods are being used. The API provides movie information in JSON format.


Postman was used to test the API. With this, I used authentication and authorization code in the form of basic HTTP authentication and JWT authentication.

Client-Side

After the API was completed, I began to develop the interface that users would need when making requests, and receiving responses, from the server-side. It is a responsive application developed using React and React-Redux. It provides the users with several interface views such as main view, single movie view with all data about a single movie in database, a login view, a registration view, and a profile view where users are able to update their information and 'favorite movies' list.

Development Challenges

Development Challenges

This has been my favorite project that was part of my web development course, and which I continue to update as I am an avid movie aficionado. I have enjoyed building, updating the API, and making sure content is updated for users to view and access. What made this project challenging was the fact that I used different tools to build the application given that my personal computer was very outdated and many compatibility issues arose that caused unforseen delays. Notwithstanding these issues, I was able to achieve the same outcome as expected using the valuable assistance from my mentor and tutor, and from reading postings on Stack Overflow and looking at similar projects found online. In the end, after some delays both my mentor and tutor were pleased with my resourcefulness in resolving the issues, and coming up with a fully-functional and aesthetically pleasing application.

Duration

The development of the client-side of this application took me nearly thrice as long as it would have normally taken because of all of the technical compatibility issues that arose during the building process. However, I found a workaround which impressed my mentor and tutor when I was finally able to submit the project for approval. Despite these challenges, I enjoyed very much working on this project as I learned a great deal about troubleshooting and about being resourceful.
