Work

myFlix Movie Application

It is a web application that provides users with access to information about movies, directors, genres, and release date. Users are able to register, update personal data, and create a list of favorite movies based on what is available in the database. The MERN stack was used to develop the server-side and the client-side.

Server-Side Technical Info

Using Node.js and Express, I created a RESTful API that interacts with a non-relational database (MongoDB). The API is accessible via HTTP methods like GET or POST. CRUD methods are used in order to retrieve data from and store data into the movie database. The API provides movie information in JSON format.

In order to test the API, I made use of Postman. Moreover, I included user authentication and authorization code in the form of basic HTTP authentication and JWT authentication.

Client-Side Technical Info

Once the API was built, I began to build the interfact that users would need when interacting with the server-side. It is a responsive application developed using React and React-Redux. It provides several views, including a main view which shows a list of all the movies in the database, single movie view which allows users to add it to their favorites list, a login view, a registration view, and a profile view where registered users are able to update their information and list of favorites.

Conclusion

This was a challenging project given the combination of different tools and technologies, but it has been my favorite project of the course so far. I have enjoyed using the Terminal while building the API and working with database structures. While developing the client-side, React and Redux worked to produce the outcome I desired for this project. With the guidance of my mentor and tutor, we overcame some technology limitations from my 4+ years-old laptop.

Native Chat Application

A chat application for use with mobile devices. Using the chat interface, users are able to send/receive text messages, images, and share their location.

Technical Info

The application was built using React Native. In order to monitor the development of this application, I used Expo.io and Android Studio. The application was optimized to function both on Android and iOS devices. Using Gifted Chat library, I created the interface and functionality of the application.

In order to store the messages, I the Google Firestore Database. For image storing, I used Google's Firebase Cloud Storage. Moreover, in order for chat conversations to be accessible when offline, I implemented asyncStorage to store the messages locally. In order to authenticate users anonymously, I used Google Firebase authentication.

Error Handling

While building the application, I ran into some version compatibility issues with my development equipment and the packages or dependencies used to develop the application. However, I adjusted the Node versions, re-installed Expo Go, and toggled some settings in order to get my application to run as expected and stable.

Conclusion

In building the Chat Application, I completed the final project required in my full-stack development course. Aside from the technical difficulties arising from compatibility issues, I realized that my problem solving skills had developed to the point of spotting some bugs before it was too late and resolved compatibility issues.

Meet Application

It is a serverless progressive web application (PWA) that was built using React and a test-driven development (TDD) approach.

Technical Info

The application uses an API to fetch upcoming event for selected cities, uses OAuth2 authentication flow. The app uses AWS Lambda for the authorization server. Also, using the OOP approach, user alerts are created.

Given the use of a service worker, the app works also while offline. Application uses tokens for authorization and access of Google Calendar API.

Moreover, in order to visualize data, I used the Recharts library to create a scatter-plot graph and a pie-chart for events fitting the search criteria.

Conclusion

While building the Meet Application, I learned quickly how to utilize the google-calendar API to generate a sample of events that could be sorted by types of event and event location.
